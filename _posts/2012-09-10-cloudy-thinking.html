Our department has a kind of standing meeting every Monday, where someone (picked at random using a Matlab script, pffft, Scientists eh? :D) gives a talk. Normally this is pretty science-y, being a research group and all - but today I gave a shortish talk on my outlook of Cloud Computing.




It's a pretty rambly affair (it was kind of chucked together on the balcony on a lovely sunny day with a beer in hand) but I guess the main things I wanted to get across were:

  • IaaS is hard, so use (mature) Infrastructure Automation tools like Chef to manage your servers as code
  • A lot of the sales bullshit about 'The Cloud' being all automatic and magic with regard to scaling and resilience is precisely that - bullshit
  • PaaS is awesome and people need to read more about how great people like Heroku) are
  • Modern Cloud providers are offering a varied and useful box of tools to build applications and organisations on top of - some better than others
  • It's mostly about letting experts get on with what they're expert at and getting things going quicker, at a scale which fits rather than a best guess


Some Links for further reading

  • The Twelve Factor App - Seriously, the best one stop shop methodology for building a thing that will run like buttered, bulletproof silk on a platform like Heroku.
  • How to build 100 web servers in a day - Great writeup by John Arundel on Infrastructure management.
  • Puppet Labs - I prefer Chef as it's more 'meta' than Puppet, but they're awesome nonetheless.Their quickstart docs are great to read.
  • Fog.rb - the premier Ruby cloud abstraction library.
  • Vagrant - Create and configure lightweight, reproducible, and portable development environments via Virtualbox and some Ruby Ninjitsu.
  • Martin Fowler's rather accurate definition of TDD.
  • Cucumber-chef gem for Ruby. Enabling test-driven-infrastructure-as-code, which is crazy but awesome! Uses the niche but handy-to-know-about LXC Linux Containers project.
